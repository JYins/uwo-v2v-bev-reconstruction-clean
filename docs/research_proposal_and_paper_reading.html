Generative V2V 3D Reconstruction

Combined proposal and paper-reading notes. Source markdown is kept at docs/source_markdown/research_proposal_generative_v2v_3d_reconstruction.md.

Current project position: the practical finished experiment is masked multi-height BEV reconstruction. The longer research direction is BEV-to-point-cloud reconstruction for communication-efficient V2V perception.

Problem Statement

Raw LiDAR sharing gives rich geometry but is too expensive for V2V communication. Box-only or highly compressed feature sharing saves bandwidth but loses geometric context. This project studies whether a compact multi-height BEV can be transmitted and then reconstructed with a lightweight generative model.

Technical Pipeline

StageDesignReason
SenderConvert LiDAR into multi-height BEV.Preserves vertical structure while becoming image-like and compact.
TransmissionSend BEV representation instead of raw point cloud.Lower bandwidth than raw LiDAR.
ReceiverUse U-Net / Pix2Pix / Diffusion to reconstruct missing BEV structure.Tests whether generative reconstruction can recover the ego blind sector.
Future perception stepConvert or use reconstruction for downstream 3D detection.Final thesis direction is task-aware perception, not just image metrics.

Strategic Decisions

  • U-Net as main baseline: fast, stable, and best in the confirmed experiments.
  • Pix2Pix as GAN comparison: useful to test adversarial sharpness, but final result is weaker than U-Net.
  • Diffusion as corrected negative baseline: v3 is implemented fairly, but over-predicts occupancy and is too slow/weak for the current task.
  • Multi-height BEV: 8 channels keep density and height information without raw point-cloud bandwidth.
  • Masked Occ-IoU: more important than full-image PSNR because the task is hidden-sector reconstruction.

Paper Reading Map

V2V / V2X Perception

  • V2V4Real: real-world V2V dataset and benchmark used by this project.
  • V2X-ViT: strong transformer-based cooperative perception baseline.
  • CoBEVT: strong cooperative BEV baseline for perception accuracy comparison.
  • V2U4Real and V2X-Radar: recent extensions showing that this dataset ecosystem is still active.

Reconstruction / Representation

  • CORE: closest conceptual reference around cooperative reconstruction, but reconstructs BEV features rather than explicit point-cloud geometry.
  • BEV-MAE: supports the idea that BEV-guided masked reconstruction can help 3D understanding.
  • CoCMT: extreme communication-efficient object-query baseline; useful contrast because this project keeps more geometric context.

Current Experiment Link

The current repo does not yet claim final BEV-to-point-cloud detection AP. The confirmed completed contribution is a clean comparison of U-Net, Pix2Pix, and corrected Diffusion v3 for masked 8-channel BEV reconstruction on V2V4Real-derived data.
